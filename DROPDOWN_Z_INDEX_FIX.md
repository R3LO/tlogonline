# Исправление отображения переключения языка под контейнером

## Проблема
Выпадающее меню переключения языка отображалось под основным контейнером страницы, что делало его недоступным для пользователя.

## Причина
Отсутствовал CSS параметр `z-index` для элементов `.dropdown-menu`, что приводило к неправильному слою наложения элементов на странице.

## Решение
Добавлен высокий `z-index: 1050` ко всем определениям `.dropdown-menu` в CSS файлах проекта:

### Измененные файлы:

1. **tlog/static/css/base.css**
   - Добавлен `z-index: 1050` к `.dropdown-menu`

2. **tlog/static/css/components.css**
   - Добавлен `z-index: 1050` к `.navbar .dropdown-menu` (2 места)
   - Добавлен `z-index: 1050` к `.dropdown-menu`
   - Добавлен `z-index: 1050` в медиа-запрос `@media (max-width: 576px)`

3. **tlog/static/css/simple-navbar.css**
   - Добавлен `z-index: 1050` к `.dropdown-menu`
   - Добавлен `z-index: 1050` в медиа-запрос `@media (max-width: 576px)`

## Технические детали

### Значение z-index: 1050
- Стандартное значение для Bootstrap dropdown меню
- Обеспечивает отображение поверх всех стандартных элементов страницы
- Совместимо с другими UI компонентами

### Медиа-запросы
- Для мобильных устройств также сохранен высокий z-index
- Обеспечивает корректное отображение на всех размерах экранов

## Результат
Переключение языка теперь корректно отображается поверх всех элементов страницы, включая основной контейнер, и доступно для взаимодействия пользователем.
